ANALYSIS SUMMARY
================

Service Layer
- Location: src/services/{module}.service.ts
- Pattern: Export {module}Service const with getAll, getById, create, update, delete
- BaseURL: http://localhost:5209/api
- Auth: JWT Bearer token auto-added from localStorage
- Error: Centralized via errorHandler.service.ts

Page Components
- Location: src/pages/{section}/{Module}Page.tsx
- State: items, loading, showModal, modalMode, selectedItem, currentPage, filters, formData
- Key Functions: loadItems(), handleOpenModal(), handleCloseModal(), handleSubmit(), handleDelete()
- UI: header, filters, table, pagination, modal

Routing
- Pattern: /section/module (e.g., /gestion/productos)
- Protected: ProtectedRoute > AppLayout > Component
- New routes: /gestion/productos, /gestion/inventarios, /gestion/calendario

Navigation
- File: src/components/layout/Sidebar.tsx
- Add to gestionMenuItems array
- Icons: shopping_cart, inventory_2, calendar_month
- No permission check needed for Gesti√≥n

Styling
- CSS Variables: --bg-primary, --text-primary, --accent-color, etc
- Classes: .btn-primary, .badge-success, .btn-icon-edit, .form-grid
- Responsive: 1024px, 768px, 480px breakpoints
- Reuse: CRMPages.css or create section-specific CSS

Types
- Entity: Full object with id, timestamps
- CreateDto: Request data without id/timestamps
- UpdateDto: Update request structure
- ListResponse: items[], totalItems, totalPages, currentPage
